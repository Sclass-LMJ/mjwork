# Clean 한전KPS 일산사업소 — 청심환 캠페인 자율수칙 공모
**프로젝트 코드명:** `cleankpsilsan`  
**기술 스택:** HTML · CSS · Vanilla JavaScript · Supabase (PostgreSQL)  
**반응형 지원:** 데스크탑 / 태블릿 / 모바일 완전 최적화

---

## 1. 서비스 개요

한전KPS 일산사업소의 "청심환 캠페인" 일환으로, 직원들이 자율수칙을 직접 공모하고 투표할 수 있는 단일 페이지 웹앱(SPA).  
별도 회원가입/로그인 없이 누구나 접근 가능하며, 관리자는 비밀번호(하드코딩)로 관리 기능에 접근한다.

---

## 2. 페이지 구성 (상단 탭 전환 방식)

| 탭 번호 | 탭명 | 주요 기능 |
|--------|------|----------|
| 1탭 | 공모 | 고정수칙 표시 + 자율수칙 제출 |
| 2탭 | 리스트 | 제출된 자율수칙 목록 + 투표 체크박스 |
| 3탭 | 통계 | 득표 순위 및 득표율 표시 |

상단 고정 헤더에 사이트명 + 탭 네비게이션 배치.

---

## 3. 페이지별 상세 명세

### 3-1. 1탭 — 공모 페이지

#### 고정수칙 (4가지, 수정 불가, 화면에 항상 표시)
1. 식사 당번제 등 직원에게 부담을 주는 관행을 운영하지 않는다.
2. 출퇴근 시 차량지원 등 불필요한 수행의전 관행을 지양한다.
3. 전용 좌석 등 관행적 특혜요소를 발굴하여 개선한다.
4. 회의는 간소화하고 효율적으로 운영한다.

#### 자율수칙 섹션
- **공모 중 안내 문구:** `자율수칙 기재 (공모중)`
- **리스트 보기 버튼:** 클릭 시 2탭(리스트)으로 이동
- **자율수칙 입력 필드:** 텍스트 입력 (placeholder: 자율수칙을 입력해 주세요)
- **제출 버튼:**
  - 클릭 시 Supabase `자율수칙` 테이블에 INSERT
  - `공개여부` 컬럼은 기본값 `Y`로 자동 저장
  - 제출자 정보 없음 (익명 무제한 제출)
  - **공고마감 상태일 경우** 버튼 비활성화 + "공모가 마감되었습니다" 안내 표시

---

### 3-2. 2탭 — 리스트 페이지

#### 목록 표시
- Supabase `자율수칙` 테이블에서 `공개여부 = 'Y'`인 항목만 표시
- 실시간 구독(Supabase Realtime)으로 신규 제출 즉시 반영
- 항목 구성: `[체크박스] [자율수칙 내용]`

#### 체크박스 동작
- **투표진행 = Y일 때만 체크박스 활성화**, 그 외에는 비활성화(disabled) 상태로 표시
- 최대 체크 가능 항목 수는 관리자 설정값 기준 (예: 2 입력 시 최대 2개)
- 최대 개수 초과 시 추가 체크 불가 + 안내 토스트 메시지 표시

#### 투표 제출 버튼 (우측 하단 고정)
- 체크한 항목들을 한 번에 Supabase `투표` 테이블에 INSERT
- **중복투표 방지:** 브라우저 로컬스토리지에 `voted: true` 저장 → 재진입 시 이미 투표한 경우 버튼 비활성화 + "이미 투표하셨습니다" 안내

#### 기명/무기명 분기
| 투표 방식 | 처리 |
|---------|------|
| 무기명 | 사번 입력 없이 바로 제출 |
| 기명 | 제출 버튼 클릭 시 사번 입력 모달 팝업 → 사번 입력 후 제출 |

---

### 3-3. 3탭 — 통계 페이지

- 누구나 접근 가능 (별도 인증 없음)
- `투표` 테이블 집계 기준으로 **득표 많은 순 정렬**
- 항목별 표시: `자율수칙 내용` / `득표 수` / `득표율(%)`
- 득표율 = 해당 항목 득표 수 ÷ 전체 득표 수 × 100
- 기명 정보(사번)는 이 페이지에 표시하지 않음 → Supabase 콘솔에서 관리자가 직접 확인

## 4. 관리자 모드

### 접근 방식
- 화면 우측 하단 플로팅 버튼 (자물쇠 아이콘 등)
- 클릭 시 비밀번호 입력 프롬프트 → `3229` 일치 시 관리자 모달 오픈
- 비밀번호는 JS 내 하드코딩

### 관리자 모달 기능 (5가지 설정)

| 설정 항목 | 타입 | 설명 |
|---------|------|------|
| 공고마감 여부 | 토글(Y/N) | Y이면 1탭 제출 버튼 비활성화, "공모가 마감되었습니다" 안내 |
| 투표진행 여부 | 토글(Y/N) | Y이면 2탭 체크박스 활성화, 투표 제출 FAB 표시 |
| 투표마감 여부 | 토글(Y/N) | Y이면 투표진행 상태와 무관하게 체크박스·제출버튼 비활성화, "투표가 마감되었습니다" 안내 |
| 투표 방식 | 라디오(기명/무기명) | 기명이면 사번 입력 모달 표시 |
| 최대 체크 가능 항목 수 | 숫자 입력 | 2탭에서 체크 가능한 최대 개수 |

> **투표진행 vs 투표마감 우선순위:**  
> 투표마감(Y)이 투표진행(Y)보다 항상 우선한다.  
> 즉, 투표진행=Y이더라도 투표마감=Y이면 투표 불가 상태로 처리한다.

- 설정 변경 시 Supabase `관리자설정` 테이블의 고정 단일 행(id=1)을 UPDATE
- 저장 버튼 클릭 시 반영, 성공 토스트 표시

## 5. Supabase 테이블 설계

### 5-1. 테이블 목록

| 테이블명 | 용도 |
|--------|------|
| `자율수칙` | 직원들이 제출한 자율수칙 저장 |
| `투표` | 투표 기록 저장 (기명/무기명) |
| `관리자설정` | 공고마감, 투표방식 등 설정값 단일 행 관리 |

---

### 5-2. SQL DDL

#### ① 자율수칙 테이블

```sql
CREATE TABLE 자율수칙 (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  내용        TEXT NOT NULL,
  공개여부     CHAR(1) NOT NULL DEFAULT 'Y' CHECK (공개여부 IN ('Y', 'N')),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 공개여부 Y인 항목만 빠르게 조회하기 위한 인덱스
CREATE INDEX idx_자율수칙_공개여부 ON 자율수칙 (공개여부);
```

#### ② 투표 테이블

```sql
CREATE TABLE 투표 (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  자율수칙_id   UUID NOT NULL REFERENCES 자율수칙(id) ON DELETE CASCADE,
  사번          TEXT,         -- 기명 투표 시 저장, 무기명이면 NULL
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 집계 성능을 위한 인덱스
CREATE INDEX idx_투표_자율수칙id ON 투표 (자율수칙_id);
```

#### ③ 관리자설정 테이블

```sql
CREATE TABLE 관리자설정 (
  id                    INT PRIMARY KEY DEFAULT 1,   -- 항상 1행만 존재
  공고마감여부           CHAR(1) NOT NULL DEFAULT 'N' CHECK (공고마감여부 IN ('Y', 'N')),
  투표진행여부           CHAR(1) NOT NULL DEFAULT 'N' CHECK (투표진행여부 IN ('Y', 'N')),
  투표마감여부           CHAR(1) NOT NULL DEFAULT 'N' CHECK (투표마감여부 IN ('Y', 'N')),
  투표방식               TEXT NOT NULL DEFAULT '무기명' CHECK (투표방식 IN ('기명', '무기명')),
  최대체크항목수          INT NOT NULL DEFAULT 1 CHECK (최대체크항목수 >= 1),
  created_at            TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at            TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 초기 설정 행 삽입 (앱 최초 실행 전 1회 실행)
INSERT INTO 관리자설정 (id) VALUES (1);

-- updated_at 자동 갱신 트리거
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_관리자설정_updated_at
BEFORE UPDATE ON 관리자설정
FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

### 5-3. Supabase RLS(Row Level Security) 정책 권고

> Supabase 프로젝트 설정에서 아래 정책을 적용하세요.  
> anon 키(공개 키)를 클라이언트에서 사용하므로 불필요한 쓰기/삭제를 차단합니다.

| 테이블 | SELECT | INSERT | UPDATE | DELETE |
|-------|--------|--------|--------|--------|
| 자율수칙 | 허용(공개여부=Y) | 허용 | ❌ 차단 | ❌ 차단 |
| 투표 | 허용 | 허용 | ❌ 차단 | ❌ 차단 |
| 관리자설정 | 허용 | ❌ 차단 | **service_role만 허용** | ❌ 차단 |

> ⚠️ 관리자설정 UPDATE는 service_role 키로만 가능하도록 설정.  
> 관리자 모달에서 설정 저장 시 Supabase Edge Function 또는 service_role 키를 환경변수로 분리하여 처리 권장.  
> (또는 간이 운영 목적이라면 anon 키에 UPDATE 허용 후 비밀번호 하드코딩으로 접근 제한)

- 배경: 딥 네이비 `#0D2B5E`
- 사이트명 텍스트: 흰색, 20px Bold
- 좌측 로고 영역에 ⚡ 아이콘 (전력 모티프) 추가
- 탭 바: 헤더 하단에 붙어있는 화이트 배경 탭
  - 비활성 탭: 슬레이트 텍스트 `#6B7C93`
  - 활성 탭: 딥 네이비 텍스트 + 하단 3px 스카이블루 언더라인 애니메이션

#### 카드 (고정수칙)
- 배경: 화이트
- 모서리: `border-radius: 12px`
- 그림자: `box-shadow: 0 2px 12px rgba(13,43,94,0.08)`
- 좌측: 숫자 배지(딥 네이비 원형) + 2px 딥 네이비 수직 라인 (시그니처 회로 라인 모티프)
- hover: 그림자 강화 + 살짝 위로 이동 (`translateY(-2px)`, transition 200ms)

#### 버튼
| 종류 | 스타일 |
|-----|-------|
| 주요 액션 (제출·투표) | 딥 네이비 배경, 흰 텍스트, radius 8px, hover 시 스카이블루로 전환 |
| 보조 액션 (리스트 보기) | 테두리만 있는 아웃라인 버튼, 딥 네이비 컬러 |
| 비활성 | 그레이 배경 `#CBD5E0`, 커서 not-allowed |

#### 체크박스 (리스트 탭)
- 커스텀 체크박스: 기본 input 숨기고 CSS로 재구현
- 미체크: 1.5px 슬레이트 테두리 원형
- 체크됨: 스카이블루 배경 + 흰색 체크마크
- 체크 시 해당 카드 전체에 스카이블루 좌측 테두리 강조 + 배경 연한 블루 `#EBF5FF`

#### 토스트 메시지
- 화면 상단 중앙 슬라이드다운 애니메이션
- 성공: 민트 그린 배경 `#00C9A7` + 흰 텍스트
- 경고/오류: 레드 배경 `#E53E3E` + 흰 텍스트
- 2.5초 후 자동 사라짐

#### 배지
| 상태 | 컬러 | 텍스트 |
|-----|------|-------|
| 공모중 | 민트 그린 | 🟢 공모중 |
| 공모마감 | 레드 | 🔴 공모마감 |
| 투표진행중 | 스카이블루 | 🔵 투표진행중 |
| 투표마감 | 레드 | 🔴 투표마감 |

> **배지 표시 우선순위 (2탭 리스트 헤더 기준):**  
> 투표마감=Y → 🔴 투표마감  
> 투표진행=Y → 🔵 투표진행중  
> 투표진행=N, 투표마감=N → 배지 미표시 (투표 예정)

#### 통계 게이지 바
- 배경 트랙: `#E2E8F0` (연한 그레이)
- 채워지는 바: 1위 딥 네이비 → 2위 스카이블루 → 3위 민트 그린 → 이후 슬레이트
- 바 너비는 CSS `width` 트랜지션으로 로드 시 0 → 실제값 애니메이션 (0.6s ease-out)

#### 관리자 플로팅 버튼
- 위치: 화면 우측 하단 고정 (`position: fixed; bottom: 24px; right: 24px`)
- 모양: 원형 44px, 딥 네이비 배경 + 자물쇠 🔒 이코
- hover: 스카이블루로 전환 + 살짝 확대 (`scale(1.1)`)

#### 관리자 모달
- 딤 처리 오버레이 (배경 `rgba(0,0,0,0.5)`) + 페이드인
- 모달 카드: 화이트, `border-radius: 16px`, 최대 너비 440px
- 각 설정 항목은 카드형 행으로 구분
- 토글 스위치: iOS 스타일 커스텀 토글 (OFF: 그레이, ON: 스카이블루)

## 8. 주요 UX 플로우 요약

```
[직원 - 자율수칙 제출]
1탭 접근 → 내용 입력 → 제출 버튼 클릭
  → 공고마감 Y? → 버튼 비활성 (제출 불가)
  → 공고마감 N? → Supabase INSERT → 성공 토스트

[직원 - 투표]
2탭 접근 → 목록 확인
  → 투표마감 Y? → 체크박스·제출버튼 비활성 + 🔴 투표마감 배지 표시 (투표진행 여부 무관)
  → 투표마감 N AND 투표진행 N? → 체크박스 비활성 (투표 예정)
  → 투표마감 N AND 투표진행 Y?
      → 로컬스토리지 voted=true? → "이미 투표하셨습니다" 안내, 버튼 비활성
      → 미투표 → 체크 (최대 N개) → 제출 버튼 클릭
          → 기명이면 사번 입력 모달 → 사번 입력 후 최종 제출
          → 무기명이면 바로 제출
      → Supabase INSERT → 로컬스토리지 voted=true 저장 → 성공 토스트

[관리자]
플로팅 버튼 → 비밀번호 입력(3229) → 관리자 모달
  → 5가지 설정 변경 → 저장 → Supabase UPDATE

[관리자 설정 상태 전이 시나리오]
공모기간:   공고마감N / 투표진행N / 투표마감N  → 자율수칙 제출 가능, 투표 불가
투표기간:   공고마감Y / 투표진행Y / 투표마감N  → 제출 불가, 투표 가능
투표종료:   공고마감Y / 투표진행Y / 투표마감Y  → 제출 불가, 투표 불가, 통계만 열람
```
