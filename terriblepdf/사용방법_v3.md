# 부가가치세 신고서 동적 파싱 시스템 V3 사용 방법

## 개요

이 시스템은 "부가세정보" 시트에 설정한 내용에 따라 동적으로 부가가치세 신고서 텍스트를 파싱합니다.

## 주요 특징

### 1. 동적 설정 기반 파싱

- "부가세정보" 시트에 항목을 추가하면 자동으로 해당 항목을 파싱
- 코드 수정 없이 설정 변경만으로 다양한 양식 처리 가능

### 2. 세율을 구분자로 사용하는 스마트 파싱

**핵심 아이디어**: 세율을 **구분자**로 사용하여 금액과 세액을 정확히 분리합니다.

데이터 형식 예시:

- **금액\_10/100\_세액**: "10/100"을 구분자로 사용
- **금액\_0/100\_세액**: "0/100"을 구분자로 사용
- **금액\_㉮\_세액**: "㉮"를 구분자로 사용
- **금액\_세액**: 세율 없음 (순차적으로 첫 번째 숫자=금액, 두 번째 숫자=세액)

### 3. 다양한 세율 형식 인식

- **분수 형식**: 10/100, 0/100
- **원문자**: ㉮, ㉯, ㉰, ㉱, ㉲, ㉳, ㉴, ㉵, ㉶, ㉷, ㉸

## 사용 방법

### Step 1: 부가세정보 시트 생성

1. Excel에서 VBA 편집기 열기 (Alt + F11)
2. 새 모듈 생성: 삽입 → 모듈
3. `changetexttoexcel_v3.bas` 파일의 내용을 붙여넣기
4. 매크로 실행: `부가세정보시트_생성` 실행

### Step 2: 부가세정보 시트 설정

생성된 "부가세정보" 시트의 각 열 설명:

| 열  | 항목명     | 설명                                               | 예시                     |
| --- | ---------- | -------------------------------------------------- | ------------------------ |
| A   | 구분       | 항목의 이름 (표시용)                               | "세금계산서 발급분"      |
| B   | 키워드1    | 찾을 키워드 앞부분 (띄어쓰기 포함 가능)            | "과세 세금계산서 발급분" |
| C   | 키워드2    | 찾을 키워드 뒷부분 (보통 순번, 띄어쓰기 포함 가능) | "(1)"                    |
| D   | 데이터형식 | 세율을 포함한 데이터 형식 (세율이 구분자 역할)     | "금액*10/100*세액"       |
| E-G | 결과       | 파싱 결과가 자동으로 기록됨                        | (자동 입력)              |

**중요**: B열과 C열을 합쳐서 찾습니다!

- 예: B2="과세 세금계산서 발급분" & C2="(1)"
- 띄어쓰기 제거 후: "과세세금계산서발급분(1)"
- 이 값으로 텍스트에서 검색합니다.

#### 설정 예시:

```
A열(구분)                      B열(키워드1)                   C열(키워드2)  D열(데이터형식)
세금계산서 발급분             과세 세금계산서 발급분        (1)           금액_10/100_세액
매입자발행 세금계산서         매입자발행 세금계산서         (2)           금액_10/100_세액
신용카드·현금영수증 발행분    신용카드ㆍ현금영수증 발행분   (3)           금액_10/100_세액
기타(정규영수증 외 매출분)    기타(정규영수증 외 매출분)    (4)           금액_세액
영세율 세금계산서 발급분      영세율 세금계산서 발급분      (5)           금액_0/100_세액
합계                          합계                          (9)           금액_㉮_세액
```

**찾는 방법:**

- B열 + C열을 합침: "과세 세금계산서 발급분" + "(1)" = "과세 세금계산서 발급분(1)"
- 띄어쓰기 제거: "과세세금계산서발급분(1)"
- 이 값으로 텍스트에서 검색

### Step 3: 텍스트 파싱 실행

1. 매크로 실행: `부가가치세신고서_동적파싱` 실행
2. 파일 선택 대화상자에서 OCR 변환된 텍스트 파일 선택
3. 파싱이 자동으로 진행됩니다.

### Step 4: 결과 확인

파싱 완료 후 두 곳에 결과가 기록됩니다:

1. **부가세신고서 시트**: 깔끔하게 정리된 최종 결과
2. **부가세정보 시트** (E-G열): 설정과 함께 결과 확인 가능

## 작동 원리

### 1. 텍스트 전처리

```
원본: "과 세   세 금 계 산 서   발 급 분   ( 1 )   5,979,793,104   10/100   597,979,306"
처리: "과세세금계산서발급분(1)5,979,793,10410/100597,979,306"
```

- 텍스트 파일의 **모든 띄어쓰기 제거**
- 설정 시트 B열&C열 합친 후 **띄어쓰기 제거**
- 양쪽 모두 띄어쓰기가 없어야 정확한 매칭 가능

### 2. 순차적 검색

- 각 항목을 처리한 후, 다음 항목은 처리된 위치 이후부터 검색
- 중복 추출 방지 및 정확도 향상

### 3. 세율 구분자 파싱 방식

#### 예시 1: "금액\_10/100\_세액"

```
설정:
  - B2="과세 세금계산서 발급분" & C2="(1)" → 검색어: "과세세금계산서발급분(1)"
  - D2="금액_10/100_세액" → 세율 구분자: "10/100"

텍스트: "...과세세금계산서발급분(1)5,979,793,10410/100597,979,306..."

처리 과정:
  1. 키워드 찾기: "과세세금계산서발급분(1)" 위치 확인
  2. 키워드 뒤 텍스트: "5,979,793,10410/100597,979,306"
  3. 세율 "10/100" 찾기 → 위치 확인
  4. 세율 앞 부분: "5,979,793,104" → 금액
  5. 세율 뒤 부분: "597,979,306" → 세액

결과:
  - 금액: 5,979,793,104
  - 세율: 10/100
  - 세액: 597,979,306
```

#### 예시 2: "금액\_세액" (세율 없음)

```
설정:
  - B2="기타(정규영수증 외 매출분)" & C2="(4)" → 검색어: "기타(정규영수증외매출분)(4)"
  - D2="금액_세액" → 세율 없음

텍스트: "...기타(정규영수증외매출분)(4)4,237,858423,784..."

처리 과정:
  1. 키워드 찾기: "기타(정규영수증외매출분)(4)" 위치 확인
  2. 키워드 뒤 텍스트: "4,237,858423,784"
  3. 세율 없음 → 순차 추출
  4. 첫 번째 숫자: "4,237,858" → 금액
  5. 두 번째 숫자: "423,784" → 세액

결과:
  - 금액: 4,237,858
  - 세율: (없음)
  - 세액: 423,784
```

#### 예시 3: 원문자 세율 "금액\_㉮\_세액"

```
설정:
  - B2="합계" & C2="(9)" → 검색어: "합계(9)"
  - D2="금액_㉮_세액" → 세율 구분자: "㉮"

텍스트: "...합계(9)25,270,541,989㉮598,403,090..."

처리 과정:
  1. 키워드 찾기: "합계(9)" 위치 확인
  2. 키워드 뒤 텍스트: "25,270,541,989㉮598,403,090"
  3. 세율 "㉮" 찾기 → 위치 확인
  4. 세율 앞 부분: "25,270,541,989" → 금액
  5. 세율 뒤 부분: "598,403,090" → 세액

결과:
  - 금액: 25,270,541,989
  - 세율: ㉮
  - 세액: 598,403,090
```

## 고급 활용

### 여러 페이지 신고서 처리

부가세정보 시트에 모든 페이지의 항목을 순서대로 나열하면 한 번에 처리됩니다.

### 새로운 양식 추가

1. 부가세정보 시트에 새 행 추가
2. 찾을 키워드와 데이터 형식 입력
3. 즉시 파싱 가능

### 키워드 작성 팁

- **B열과 C열을 합쳐서** 검색하므로 두 열을 적절히 나누어 입력
  - 보통 B열: 본문 키워드, C열: 순번
  - 예: B="과세 세금계산서 발급분", C="(1)"
- 띄어쓰기 포함 여부는 상관없음 (자동으로 제거됨)
- 특수문자(괄호, 하이픈 등)는 원본 그대로 입력
- B열&C열을 합쳤을 때 중복되지 않는 고유한 값이 되도록 설정

### 데이터 형식 작성 팁

- **D열에는 실제 세율 값을 포함**하여 입력
  - "금액*10/100*세액" ← 10/100이 구분자
  - "금액*0/100*세액" ← 0/100이 구분자
  - "금액*㉮*세액" ← ㉮가 구분자
  - "금액\_세액" ← 세율 없음
- 세율이 텍스트에서 정확히 나타나는 형태로 입력
- 세율이 여러 형태면 각각 다른 행으로 설정

## 주의사항

1. **B열&C열 합침**: B열과 C열을 합쳐서 검색합니다. 띄어쓰기는 자동 제거됩니다.
2. **키워드 고유성**: B열&C열을 합친 값이 중복되지 않도록 설정 (첫 번째만 처리됨)
3. **순서 중요**: 부가세정보 시트의 행 순서대로 처리되므로, 신고서 순서대로 나열
4. **세율이 구분자**: D열에 입력한 세율이 금액과 세액을 구분하는 역할을 합니다.
5. **세율 정확성**: D열의 세율이 텍스트에 정확히 있어야 합니다.
   - 텍스트에 "10/100"이 있으면 D열에도 "금액*10/100*세액"
   - 텍스트에 "㉮"가 있으면 D열에도 "금액*㉮*세액"

## 문제 해결

### Q: 특정 항목이 추출되지 않습니다.

A:

- **B열&C열을 합친 값**이 정확한지 확인
  - 예: B="과세세금계산서발급분" & C="(1)" → "과세세금계산서발급분(1)"
- 원본 텍스트 파일에서 해당 키워드가 있는지 확인 (띄어쓰기 제거 후)
- 데이터 형식이 올바른지 확인
- 이전 항목과 키워드가 중복되지 않는지 확인

### Q: 금액이 0으로 나옵니다.

A:

- 키워드가 텍스트에서 정확히 찾아지는지 확인
- D열의 세율 값이 텍스트에 정확히 있는지 확인
- 세율을 구분자로 사용하므로, 세율이 없으면 찾을 수 없습니다
- 세율이 없는 경우 D열을 "금액\_세액"으로 설정

### Q: 세율이 추출되지 않습니다.

A:

- D열에 세율을 명시했는지 확인
  - 예: "금액*10/100*세액" (O) vs "금액*세율*세액" (X)
- 텍스트에 있는 세율 형식과 D열의 세율이 정확히 일치하는지 확인
  - 텍스트: "10/100" → D열: "금액*10/100*세액"
  - 텍스트: "㉮" → D열: "금액*㉮*세액"

### Q: 세액이 0으로 나옵니다.

A:

- D열의 세율 값이 텍스트에서 정확히 찾아지는지 확인
- 세율 뒤에 숫자가 바로 나오는지 확인
- 세율과 세액 사이에 다른 문자가 있으면 추출 실패

### 왜 세율을 구분자로 사용하나요?

OCR 텍스트에서는 띄어쓰기가 모두 제거되어 숫자들이 붙어있습니다:

```
"5,979,793,10410/100597,979,306"
```

이 경우 어디까지가 금액이고 어디서부터가 세액인지 알 수 없습니다.
하지만 **세율("10/100")을 구분자로 사용**하면:

```
"5,979,793,104" | "10/100" | "597,979,306"
    금액             세율          세액
```

명확하게 분리할 수 있습니다! 🎯
